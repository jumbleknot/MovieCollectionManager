# Contract: Env-Var Manifest (`<stack>.env.example` ↔ generator ↔ `<stack>.env`)

This is the contract between the committed placeholder templates, the generator, and the gitignored output. The full variable list lives in [../data-model.md](../data-model.md); this defines the *format and behavior* contract.

## Template file format (`<stack>.env.example`, committed)

- Plain dotenv: one `NAME=VALUE` per line; `#` comment lines and blank lines allowed.
- **No inline comments on value lines** (project rule — dotenv loaders treat everything after `=` as the value).
- `VALUE` is one of:
  - `<generate:KIND>` — a placeholder the generator replaces (`KIND` ∈ `b62-32`, `b62-48`, `hex-64`, `complex-16`, `unleash-admin`, `unleash-client`).
  - a literal — a deterministic fixture, copied verbatim (e.g. `LANGFUSE_INIT_PROJECT_PUBLIC_KEY=pk-lf-mcm-dev-0000000000000000`).
- A header comment block documents the stack and that real values are generated, never committed.

## Generator contract (`scripts/gen-dev-secrets.mjs`)

**Invocation**: `node scripts/gen-dev-secrets.mjs [--force] [--stack=auth|mcm|audit|observability]`

**Behavior**:
1. For each in-scope stack, read `infrastructure-as-code/docker/stacks/<stack>.env.example`.
2. If `<stack>.env` exists and `--force` not given → **skip** that stack (idempotent; preserves a running dev's values). Log `skipped (exists)`.
3. Otherwise produce `<stack>.env` with: every `<generate:KIND>` → a freshly minted value of that KIND; every literal → copied verbatim.
4. Generated value rules per KIND — see [research R4](../research.md). Invariants the generator MUST hold:
   - `b62-*` values match `^[A-Za-z0-9]+$` (URL-safe, shell-safe) at the stated length.
   - `hex-64` matches `^[0-9a-f]{64}$`.
   - `complex-16` is 16 chars containing ≥1 each of upper/lower/digit/special, drawn from a charset excluding `@ : / ? # % & " ' \` and whitespace (safe in `-u user:pass` and YAML).
   - `unleash-admin` = `*:*.` + `b62-40`; `unleash-client` = `default:development.` + `b62-40`.
5. Exit 0 on success; non-zero on a missing/malformed template or unwritable output.

**Idempotency / rotation**: default run never overwrites; `--force` regenerates (the rotation path).

**Output file**: written with a leading comment `# GENERATED by scripts/gen-dev-secrets.mjs — DO NOT COMMIT (gitignored).`

## Output file (`<stack>.env`, gitignored)

- Matches every key in the template; values are real.
- Excluded from VCS by `.gitignore` (`*.env`); the `!infrastructure-as-code/docker/stacks/*.env.example` carve-out keeps templates tracked.
- Consumed by Compose interpolation via the stack's `include` `env_file:` and/or the Nx target `--env-file` (see [research R1](../research.md)).

## Acceptance (maps to spec)

- FR-004/FR-005: every randomized var differs run-to-run under `--force`; fixtures are stable. (verify: diff two `--force` runs — randomized lines differ, fixture lines identical.)
- FR-006: `git status` shows the `*.env.example` tracked and the `*.env` ignored. (verify: `git check-ignore stacks/auth.env` → match; `git ls-files stacks/auth.env.example` → listed.)
- AC2: second run without `--force` leaves `<stack>.env` byte-identical. (verify: hash before/after.)
