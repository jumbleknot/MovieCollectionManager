# Phase 1 Data Model: Expo SDK 55 → 56 Upgrade

## Domain entities

**None.** This feature changes the technology baseline and documentation only. It introduces, modifies, and removes zero domain entities, value objects, aggregates, relationships, validation rules, or state transitions. The MongoDB `movie_collections` / `movies` schemas, the BFF `Session` model, and all DTOs are unchanged.

Because there is no data model to design, the rest of this document instead inventories the **configuration and version artifacts** the upgrade mutates — the "entities" that actually change — so the tasks phase has a concrete change surface.

## Invariants the upgrade must preserve (verification anchors, not data)

- Auth: Authorization Code + PKCE via BFF; HttpOnly/SameSite=Strict cookie; opaque session ID only client-side.
- Session: server-side Redis sessions; logout terminates BFF + IAM SSO session; refresh-token rotation.
- Access control: centralized (BFF `requireMcUser`; mc-service Tower layer) — unchanged.
- API contracts: all BFF routes and mc-service `/api/v1/*` endpoints behave identically.
- Test type integrity: no mocks introduced into `tests/integration/`.
